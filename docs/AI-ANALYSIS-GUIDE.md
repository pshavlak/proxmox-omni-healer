# AI Analysis Guide - Критичность ошибок

Этот документ описывает логику анализа ошибок AI помощником и классификацию по критичности.

## Система классификации критичности

### 🔴 CRITICAL (Критично)
**Требует немедленного действия. Влияет на функциональность системы.**

- **Out of Memory (OOM)**: Контейнер может упасть
- **Disk Full**: Система не может писать данные
- **Service Failures** (критичные сервисы): Основные приложения не работают

### 🟠 MEDIUM (Среднее)
**Требует внимания. Может привести к проблемам в будущем.**

- **Connection Refused**: Сервисы не могут подключиться друг к другу
- **Service Failures** (некритичные): Вспомогательные сервисы не работают

### 🟢 LOW (Низкое)
**Информационно. Не влияет на функциональность.**

- **systemd-networkd-wait-online timeout**: Известная проблема LXC контейнеров

## Анализируемые ошибки

### 1. systemd-networkd-wait-online Timeout

**Критичность**: 🟢 LOW

**Описание**:
- Происходит при операциях `apt` в LXC контейнерах
- `systemd-networkd-wait-online` ждёт полной сетевой готовности
- В LXC эта готовность может никогда не наступить (особенность контейнеризации)
- Сетевое подключение работает нормально (eth0 имеет IP адрес)

**Признаки в логах**:
```
systemd-networkd-wait-online[PID]: Timeout occurred while waiting for network connectivity.
apt-helper: E: Sub-process /lib/systemd/systemd-networkd-wait-online returned an error code (1)
```

**Рекомендуемое исправление**:
```bash
# Отключить проверку сетевой готовности
systemctl disable systemd-networkd-wait-online.service
systemctl mask systemd-networkd-wait-online.service

# Проверить, что сеть работает
ip addr show eth0
```

**Почему это не критично**:
- Сеть работает нормально (IP адрес присвоен)
- Влияет только на скорость операций `apt`
- Приложение функционирует без проблем
- Это известная особенность LXC, а не ошибка конфигурации

**Пример из audiobookself (ID 106)**:
- Контейнер работает: ✅ running
- Приложение запущено: ✅ /usr/share/audiobookshelf/audiobookshelf (PID 85)
- Сеть работает: ✅ eth0 UP, IP 192.168.1.10/24
- Ошибка в логах: ⚠️ systemd-networkd-wait-online timeout (не влияет на работу)

---

### 2. Out of Memory (OOM)

**Критичность**: 🔴 CRITICAL

**Описание**:
- Контейнер исчерпал выделенную память
- Может привести к краху приложения

**Рекомендуемое исправление**:
```bash
# Проверить использование памяти
free -h

# Увеличить лимит памяти контейнера в Proxmox
# или оптимизировать приложение
```

---

### 3. Disk Full

**Критичность**: 🔴 CRITICAL

**Описание**:
- На диске закончилось место
- Система не может писать данные

**Рекомендуемое исправление**:
```bash
# Проверить использование диска
df -h

# Найти крупные файлы
du -sh /* | sort -h

# Освободить место (удалить логи, кэш и т.д.)
```

---

### 4. Connection Refused

**Критичность**: 🟠 MEDIUM

**Описание**:
- Сервис не может подключиться к другому сервису
- Может быть проблема с портами или сетевой конфигурацией

**Рекомендуемое исправление**:
```bash
# Проверить открытые порты
netstat -tlnp

# Проверить статус сервисов
systemctl status <service_name>
```

---

### 5. Service Failures

**Критичность**: 🟠 MEDIUM (или 🔴 CRITICAL для критичных сервисов)

**Описание**:
- Системный сервис не запустился или упал

**Рекомендуемое исправление**:
```bash
# Показать все упавшие сервисы
systemctl --failed

# Посмотреть детали ошибки
journalctl -xe

# Перезапустить сервис
systemctl restart <service_name>
```

## Как AI помощник использует эту информацию

1. **Анализирует логи** контейнера
2. **Классифицирует ошибки** по критичности
3. **Предлагает команды** для исправления
4. **Показывает пометку критичности** в UI:
   - 🔴 CRITICAL → красный цвет, требует немедленного действия
   - 🟠 MEDIUM → жёлтый цвет, требует внимания
   - 🟢 LOW → зелёный цвет, информационно

**Логи содержат**:
```
systemd-networkd-wait-online[1994]: Timeout occurred while waiting for network connectivity.
```

**AI анализирует**:
- ✅ Сеть работает (eth0 UP, IP присвоен)
- ✅ Приложение запущено (audiobookshelf процесс активен)
- ⚠️ Ошибка в логах (systemd-networkd-wait-online timeout)

**Вывод**:
- **Критичность**: 🟢 LOW
- **Рекомендация**: Отключить systemd-networkd-wait-online (опционально)
- **Действие**: Не требуется срочное вмешательство

Если нужно добавить новый тип ошибки:

1. Отредактируйте `backend/app/ai_agent.py` метод `analyze_logs()`
2. Добавьте проверку в формате:
   ```python
   if "error_pattern" in logs:
       issues.append("Описание ошибки")
       criticality_map["error_key"] = "CRITICAL|MEDIUM|LOW"
       commands.append("# Команда для исправления")
   ```
3. Обновите этот документ с описанием новой ошибки
4. Закоммитьте изменения
